Задачи на проекте

Сейчас на складах без WMS в РО подбирается серия товара по FEFO. Однако, существует ряд ТТ, которым необходимо отгружать товар с ограничением срока годности.

Данные ограничения сообщает Отдел продаж, и Сотрудники склада вынуждены знать эти ограничения наизусть. Физически подбирать товар с нужными сроками годности, затем с помощью Отдела координации переподбирают в РО нужные серии.

В итоге возникают ошибки:

  1. Кладовщик забыл, не знал, перепутал какой срок нужен в эту ТТ и собрал тот срок, который указан в напечатанном РО => это возврат
  2. Кладовщик отобрал нужный срок, но не указал в РО; не переподобрали в РО=> расхождения по сериям между ЕРП и фактом=> не верная оценка остатков на складах по срокам годности, не корректная информация по ТСКГ.

1.       Добавить в ЕРП возможность устанавливать для ТТ отсечку с минимальным сроком годности (в днях) товара для отгрузки.

2.       Добавить для ТТ Настройку запаса срока годности, в которой можно выбрать проект, с большим приоритетом, чем настройки запаса срока годности для ТТ

3.       Подбирать автоматически в РО ту серию, которая удовлетворяет условиям отсечки, но по FEFO.

1.1        Критерии успешности результата разработки (как вы предлагаете проверять реализацию каждого бизнес-требования?):

1 . В ЕРП Есть возможность настроить у ТТ (Контрагента) отсечку по минимальным срокам годности товара для отгрузки.

2. При создании РО к товару подбирается серия со сроками годности больше, чем отсечка у ТТ (Контрагента), но по FEFO.
